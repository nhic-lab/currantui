# Typography

CurrantUI self-hosts a single variable family — Source Sans 3 — via
Fontsource, no CDN requests. Both `--font-sans` and `--font-heading` resolve
to it; `font-heading` remains the semantic hook for titles, and the full
variable weight range keeps `font-medium` and `font-semibold` distinct.

## Scale

The scale is a productive dashboard ramp: default UI text sits at 14px, with
a 12px floor for meta and helper text — compact, not cramped.

| Role | Class | Notes |
|---|---|---|
| Default UI text | `text-sm/relaxed` | Buttons, inputs, table cells |
| Meta / helper floor | `text-xs` | Helper text, captions, table headers — never smaller in prose |
| Avatar internals | `text-2xs` | Size-constrained fallback initials/badges only, never general-purpose |
| Card / section titles | `text-base` | `font-heading`, sits under page titles |
| Page titles | `text-xl` | Nothing in chrome exceeds this |
| Numeric / meta | `tabular-nums` | IDs, counts, timestamps |

Arbitrary bracket font sizes (`text-[0.6875rem]` and the like) are banned in
component source — every size comes from the named scale above, checked by
grep over `src/components` (stories are exempt; they may size demo fixtures
freely). The one exception is the brand lockup in `hic-logo.tsx`, which sizes
off the logo's own proportions, not prose.

A few table surfaces are called out here since they read as inconsistent at
a glance but are deliberate: the table toolbar's search input and the table
selection bar's Clear button sit at `text-xs` (a density exception below the
default), while the table empty-state body stays at the plain default
`text-sm`. Don't "fix" any of these to match one another.

## Density

Control heights follow the compact scale: `h-5` (xs, badge), `h-6` (sm),
`h-7` (**default** for buttons, inputs, selects), `h-8` (lg, tabs list).
Default padding is `px-2` with `gap-1` between icon and label. shadcn's
stock `h-9`/`h-10` sizes are always reduced when scaffolding new components.

## Lists

There is no List component — prose lists are a class recipe on native
elements:

```tsx
<ul className="list-disc ps-4 text-sm/relaxed marker:text-muted-foreground [&_li]:mt-1">
  <li>Weekly surveillance</li>
  <li>
    Monthly service report
    <ul className="list-[circle] ps-4">
      <li>OPD visits</li>
    </ul>
  </li>
</ul>
```

<ul className="list-disc ps-4 text-sm/relaxed marker:text-muted-foreground [&_li]:mt-1">
  <li>Weekly surveillance</li>
  <li>
    Monthly service report
    <ul className="list-[circle] ps-4">
      <li>OPD visits</li>
    </ul>
  </li>
</ul>

Ordered lists swap `list-disc` for `list-decimal` (numbers stay
`tabular-nums` automatically via the token setup). For data rows use
`StructuredList`; for term/value pairs use `LabeledValue`.

## Radius

`--radius` is `0.25rem` with derived steps mapped through `@theme inline`.
Use named utilities — `rounded-md` for controls, `rounded-sm` for xs,
`rounded-lg` for containers, `rounded-full` for pills — never literal pixel
values.
