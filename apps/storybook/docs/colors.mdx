# Colors

Every color in CurrantUI is a semantic token defined as an oklch CSS variable
in both `:root` (light) and `.dark`. Components use tokens exclusively —
`bg-primary`, `border-border`, `text-muted-foreground` — never palette
utilities or hex/oklch literals.

Hover and active states shift alpha on the *same* token (`bg-primary/80`),
never switch tokens.

## Status colors

`destructive`, `success`, `warning`, and `info` share one recipe: the token is
the text color, backgrounds are alpha tints of the same token — `bg-success/10`
resting, `/20` hover (dark: `/20` → `/30`). All four are AA-checked for exactly
that pairing in both palettes. Components expose them as variants under the
same names (`success`, `warning`, `info`) — never synonyms.

## Rules

- **Semantic tokens only.** If a color isn't a token, it doesn't ship.
  Inline-SVG illustrations are the exception; prefer `currentColor` there.
- **Both palettes always.** Every token is defined in `:root` and `.dark`;
  design dark-first and verify in light.
- **Theming is token override.** Consumers re-brand by overriding variables
  after importing `globals.css` — components never change.
